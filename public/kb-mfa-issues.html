MFA Issues

Lost device, authenticator reset, switching methods, and emergency bypass.

Reset MFA for remote users

Use backup verification and reset the authenticator app.

  1. Verify identity with backup code or SMS.
  2. Remove old authenticator binding in IdP.
  3. Send new enrollment link; confirm push works.

Switch MFA method

SMS ↔ App ↔ Token.

  1. Capture requested method and justification.
  2. Update MFA factors in IdP; disable unused factors.
  3. Re-test login and record method change.

Emergency bypass

Time-boxed access with audit trail.

  1. Manager approval + ticket reference.
  2. Issue 1-time bypass code (expires in 15–30 mins).
  3. Document issuance; schedule MFA re-enroll.

Common errors

“Push denied”, “Device not registered”, “Rate limited”.

  • Check time sync on device
  • Clear app cache / reinstall
  • Confirm network/VPN not blocking push
